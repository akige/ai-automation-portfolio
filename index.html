Case studies

All systems below are my own production infrastructure. Details are anonymized; architecture walk-throughs available on a call.

CASE 01 · SELF-HEALING OPERATIONS

LLM-triaged alerting platform that fixes itself first

A central alert pipeline ingests email, probe, and heartbeat signals from a multi-region fleet, classifies severity with an LLM, auto-remediates known failure classes (service restarts, disk cleanup, stale-data refresh), and only escalates genuine P0/P1 incidents to a human — with a human-in-the-loop web UI for everything ambiguous.

  • Watchdogs with two-strike confirmation gates to kill false-positive pages
  • Every remediation logged with verify + rollback steps, auditable per incident
  • Stack: Python, FastAPI, systemd, IMAP ingestion, LLM gateway with multi-model routing

CASE 02 · AUTONOMOUS CODE-AND-VERIFY

Live-stream pipeline: audience request → AI writes code → headless browser verifies → auto-publish

For a live coding-stream format, viewer requests are bridged from chat to a coding agent that builds the requested mini-app on the spot. A headless browser then runs acceptance checks against the artifact before it is published to a public gallery — fully unattended, with monitoring and auto-restart around every leg of the pipeline.

  • End-to-end: chat ingestion → agent execution → E2E acceptance → publish
  • Guardrail hooks block credential/PII exposure before anything goes on screen
  • Stack: Node.js, WebSocket bridge, Claude-based coding agent, Playwright-style headless verification

CASE 03 · MULTI-AGENT ORCHESTRATION

Coordination layer for 50+ concurrent AI sessions without collisions

When many AI agents work the same infrastructure, they overwrite each other. I built a coordination service — atomic compare-and-swap claims, leases with TTL, fencing tokens, and a cross-session handoff inbox — so dozens of concurrent agent sessions share machines, files, and tasks safely. Compliance hooks enforce documentation and changelog discipline automatically.

  • Atomic task claiming: no duplicated work, no clobbered files, audit trail per session
  • Knowledge base + RAG so every new agent session cold-starts with full operational context
  • Stack: Python, CAS/lease service, git-backed knowledge vault, embedding search
